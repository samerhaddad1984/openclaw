Module 1 — Your Role as a LedgerLink Technician

Field Technician
▾

1 What Your Job Is

As a LedgerLink Field Technician, your role is to install, configure, train, and hand off the LedgerLink AI accounting system at CPA firm client sites. You are the face of LedgerLink for the client.

  • Install LedgerLink on the client's server computer (Windows or Mac)
  • Run the setup wizard and configure all integrations
  • Set up Cloudflare tunnel for remote access
  • Train the CPA firm owner and staff on basic usage
  • Verify everything works before leaving the site
  • Document the installation and file a site visit report

2 What You Are NOT Responsible For

  • Making accounting decisions or giving tax advice
  • Modifying GL account charts or tax codes for the client
  • Writing or modifying Python code in production
  • Accessing client financial data beyond what is needed for testing
  • Promising features or timelines not approved by management
Never give accounting or tax advice. If a client asks about accounting treatment, direct them to their CPA or the LedgerLink documentation.

3 Tools You Need for Every Site Visit

Laptop with LedgerLink installer USBRun installation if internet is slow; carry installer files
Admin credentials (for your LedgerLink admin portal)Access provisioning system, generate keys if needed
License key for the client (printed or in email)Activate the client's license during setup
Ethernet cableBackup internet connection method
Printed site visit checklist formDocument every step; required for your report

4 Escalation Path

You fix: installation errors, port conflicts, Python version issues, Cloudflare setup, basic network config, service restarts, running autofix.py

Escalate to Senior Support: database corruption, license signing issues, API integration failures that persist after basic troubleshooting, custom code modifications, hardware failures

When escalating, always collect: version number, autofix.py output, last 50 lines of install.log, database size, and exact error message with steps to reproduce.

5 The 5-Step Site Visit Checklist

  1. Assess: Verify system requirements, internet, available ports, and admin access
  2. Install: Run the installer, complete setup wizard, configure Cloudflare
  3. Verify: Run autofix.py, test all features, verify remote access
  4. Train: Walk the owner and staff through basic operations
  5. Handoff: Print credentials, leave documentation, file site visit report

Module 2 — LedgerLink Architecture for Technicians

Field Technician
▾

1 How the Software Works

LedgerLink is a Python HTTP server that runs on a single computer at the CPA firm. It uses an SQLite database to store all data and runs as a Windows Service (or launchd on Mac) so it starts automatically on boot.

  • The server is written in Python using only the standard library HTTP server plus a few dependencies (bcrypt, pdfplumber, Pillow, reportlab, etc.)
  • All business logic runs on the server — browsers are just the display layer
  • AI features (document classification, GL suggestions) call external APIs (DeepSeek, Anthropic Claude)

Port 8790 (setup wizard) is only used during initial installation. After setup is complete, only 8787 and 8788 need to be running.

4 The Cloudflare Tunnel

A Cloudflare tunnel creates a secure connection from the client's server to the Cloudflare network, allowing remote access without opening ports on the router.

  • Required for: remote access to the dashboard, client portal access from outside the office, remote support by technicians
  • The tunnel exposes localhost:8787 (dashboard) and localhost:8788 (client portal) through a public HTTPS URL
  • Runs as a Windows Service (cloudflared) or launchd job on Mac
  • Free tier from Cloudflare is sufficient

Module 4 — Windows Installation Step by Step

Field Technician
▾

1 Copy Installer Files

Copy the entire LedgerLink installer folder from the USB drive to the client's Desktop. The folder should contain: INSTALL.bat, requirements.txt, scripts/, src/, installer/, and all other project files.

2 Run INSTALL.bat as Administrator

Right-click INSTALL.bat → Run as administrator. This is mandatory — the installer will not work without admin privileges.

ALWAYS run as Administrator. The installer checks for admin privileges. If not elevated, it will attempt to re-launch with elevation.

3 What Happens During Installation

The installer runs 8 steps in sequence:

  1. [1/8] Check Python — Verifies Python 3.11+. Downloads and installs if missing.
  2. [2/8] Install dependencies — Runs pip install -r requirements.txt
  3. [3/8] Database migration — Runs migrate_db.py to create all tables
  4. [4/8] Install Windows Service — Registers LedgerLinkAI service
  5. [5/8] Start service — Starts the LedgerLink background service
  6. [6/8] Create shortcuts — Desktop shortcuts for dashboard and wizard
  7. [7/8] Start setup wizard — Launches the wizard on port 8790
  8. [8/8] Open browser — Opens http://127.0.0.1:8790/ in the default browser

4 Reading install.log

If anything fails, check the installation log:

type C:\LedgerLink\install.log

The log shows every step with timestamps. Look for lines containing ERROR or FAIL.

5 Verify the Windows Service

Open Services console:

services.msc

Find "LedgerLink AI" in the list. Status should be Running and Startup Type should be Automatic.

6 Verify Ports Are Listening

netstat -ano | findstr :8787 netstat -ano | findstr :8788

Both ports should show LISTENING state.

7 Open the Setup Wizard

Navigate to:

http://127.0.0.1:8790/

Module 7 — Cloudflare Tunnel Setup

Field TechnicianSenior Tech
▾

1 Why Cloudflare Is Required

Without a Cloudflare tunnel, the dashboard and client portal are only accessible from the office local network. The tunnel provides:

  • Secure HTTPS access from anywhere (home, mobile, client sites)
  • Remote support access for technicians
  • Client portal access for document uploads from outside the office
  • No need to configure router port forwarding

2 Creating a Cloudflare Account

Go to dash.cloudflare.com and create a free account. You need a domain added to Cloudflare (the client's domain or a shared LedgerLink domain).

3 Installing cloudflared

Windows:

winget install Cloudflare.cloudflared

Mac:

brew install cloudflared

4 Creating and Configuring the Tunnel

You can use the automated script:

python scripts/setup_cloudflare.py

Or manually:

# Authenticate with Cloudflare cloudflared login # Create the tunnel cloudflared tunnel create ledgerlink-firmname # Route DNS cloudflared tunnel route dns ledgerlink-firmname portal.firmname.com

5 Register as Windows Service

cloudflared service install

This ensures the tunnel starts automatically on boot. Verify:

sc query cloudflared

6 Verify the Tunnel

Open the public URL in a browser (from your phone, not on office WiFi) to confirm it resolves to the LedgerLink dashboard.

7 Troubleshooting Tunnel Issues

IssueFix
No connectionCheck sc query cloudflared. Restart: sc stop cloudflared && sc start cloudflared
DNS not resolvingWait 5 minutes for DNS propagation. Check Cloudflare dashboard DNS records.
Certificate errorsRe-run cloudflared login to refresh certificate.

Module 9 — Generating and Activating License Keys

Field TechnicianSenior Tech
▾

1 How License Keys Work

License keys are generated on the administrator's machine using the provisioning system. They are HMAC-SHA256 signed and encode the tier, firm name, max clients, max users, and expiry date.

Key format: LLAI- followed by base64url-encoded JSON payload.

2 The 4 License Tiers

TierMax ClientsMax UsersKey Features
Essentiel103Basic review, basic posting
Professionnel305+ AI router, bank parser, fraud detection, RQ, time tracking, month-end
Cabinet7515+ Analytics, Microsoft 365, filing calendar, client communications
EntrepriseUnlimitedUnlimited+ Audit module, financial statements, sampling, API access

3 Activating a License at the Client Site

  1. During setup wizard Step 3, paste the LLAI- key
  2. The system validates the signature and checks expiry
  3. Tier, expiry, max clients, and max users are displayed
  4. Click Activate to save to ledgerlink.config.json

Or via the dashboard: go to /license → paste key → Activate.

4 Multi-Machine Licensing

Each license supports up to 3 machines. When activated on a new machine, the machine ID (based on hostname + disk serial) is registered in the database. If the limit is exceeded, a warning appears but the system continues to function.

5 What to Do If Activation Fails

  • Invalid signature: The signing secret in .env may not match. Contact the administrator.
  • Expired: The license has passed its expiry date. A new key must be generated.
  • Internet required: Activation itself does not require internet, but if the signing secret is missing, check the .env file.

Module 12 — Configuring AI Providers

1 Why AI Providers Are Needed

LedgerLink uses external AI APIs to classify documents, extract vendor names, suggest GL accounts, detect anomalies, and generate compliance narratives. Without API keys, the system falls back to keyword-only matching (less accurate).

2 The Two Providers

ProviderUsed ForCost Estimate
DeepSeekRoutine tasks: classify documents, extract vendors, suggest GL codes~$5/month for 50 clients
Anthropic ClaudeComplex tasks: explain anomalies, compliance narratives, escalation decisions~$10/month for 50 clients

3 Where to Enter Keys

Option 1: Setup wizard Step 4 (AI Providers)

Option 2: Edit ledgerlink.config.json directly:

{ "ai_router": { "routine_provider": { "api_key": "sk-your-deepseek-key-here" }, "premium_provider": { "api_key": "sk-ant-your-anthropic-key-here" } } }
After editing the config file, restart the service for changes to take effect.

4 Testing

Use the Test button in the wizard, or upload a receipt and verify that AI suggestions appear (GL account, tax code, vendor name).

5 No API Key?

If no API keys are configured, the system still works but uses keyword-only matching for document classification. AI suggestions will not appear. This is acceptable for initial setup if the client plans to add keys later.

A What the CPA Firm Needs

NOTHING. No Twilio account. No WhatsApp Business account. No API keys.

The CPA firm only needs:

  • LedgerLink running with Cloudflare tunnel active
  • You (the technician) to email support@ledgerlink.ca to activate their firm

B How It Works (Simple Explanation)

  1. Samer runs OpenClaw at openclaw.io — this is the WhatsApp/Telegram gateway
  2. When a client sends a photo to the LedgerLink WhatsApp number, OpenClaw receives it
  3. OpenClaw forwards it to the CPA firm’s LedgerLink at: https://[cloudflare-url]/ingest/openclaw
  4. LedgerLink runs OCR and creates the document in the review queue
  5. The accountant reviews it like any other document

LedgerLink never talks to WhatsApp directly. OpenClaw handles everything.

C Technician Setup Steps (Exactly 4 Steps)

Step 1: Verify Cloudflare tunnel is running and note the firm’s public URL

Example: https://tremblay-cpa.ledgerlink.app

Test it: open the URL on your phone — dashboard should load

Step 2: Email support@ledgerlink.ca with:

Subject: Activate OpenClaw for [Firm Name] Firm name: Tremblay CPA Inc Cloudflare URL: https://tremblay-cpa.ledgerlink.app Contact person: Sam Tremblay Requested services: WhatsApp / Telegram / both

Step 3: Samer activates the firm in OpenClaw (within 24 hours)

You receive a confirmation email when active. The confirmation includes the WhatsApp number clients use to send documents.

Step 4: Register each client’s phone number in LedgerLink

  • Go to dashboard → Users → find the client account
  • Enter their WhatsApp number in format +15141234567 (with country code)
  • For Telegram: enter their numeric Telegram ID
  • (Client finds their Telegram ID by messaging @userinfobot on Telegram)

D Testing After Activation

  1. Make sure Cloudflare tunnel is running
  2. Send a photo of any receipt to the LedgerLink WhatsApp number
  3. Within 30 seconds check the review queue — the document should appear
  4. If it does not appear, check the messaging log:

E Common Problems and Fixes

Problem 1: Document not created after sending WhatsApp photo

  • Cause: Client phone number not registered in LedgerLink
  • Fix: Go to Users → find client → add their WhatsApp number with country code
  • Check:
SELECT username, whatsapp_number FROM dashboard_users WHERE whatsapp_number IS NOT NULL;

Problem 2: messaging_log shows unknown_sender

  • Cause: Phone number in database does not match what WhatsApp sends
  • Fix: Check exact format — must include country code. +15141234567 not 5141234567
  • Fix:
UPDATE dashboard_users SET whatsapp_number='+15141234567' WHERE username='client_abc';

Problem 3: Cloudflare URL not reachable

  • Cause: Tunnel not running
  • Fix: sc start cloudflared (Windows) or launchctl start com.cloudflare.cloudflared (Mac)
  • Check: curl https://[cloudflare-url]/ingest/openclaw — should return 400 (not 404 or timeout)

Problem 4: Client sends text message, no document created

  • This is correct behavior — text-only messages return status: no_file
  • Tell client: always send a PHOTO or PDF, not just text

Problem 5: OpenClaw not yet activated for this firm

  • Cause: You forgot to email support@ledgerlink.ca
  • Fix: Email now with firm name and Cloudflare URL

Module 19 — Common Installation Problems and Solutions

Field TechnicianSenior Tech
▾

1 Python Not Found During Install

Symptom: INSTALL.bat shows "Python 3.11+ not found"

Cause: Python not installed or not in PATH

:: Download and install Python from python.org :: CRITICAL: Check "Add python.exe to PATH" during install :: After install, open a NEW command prompt: python --version

2 pip Install Fails with SSL Error

Symptom: SSL: CERTIFICATE_VERIFY_FAILED

Cause: Corporate firewall or proxy intercepting SSL

python -m pip install --upgrade pip --trusted-host pypi.org --trusted-host files.pythonhosted.org python -m pip install -r requirements.txt --trusted-host pypi.org --trusted-host files.pythonhosted.org

3 Port 8787 Already in Use

Symptom: "Address already in use" error

:: Find the process netstat -ano | findstr :8787 :: Kill it (replace PID) taskkill /PID 1234 /F

4 Dashboard Shows Blank White Page

Symptom: Browser shows empty page at http://127.0.0.1:8787/

Cause: JavaScript error or missing static files

:: Try running the dashboard manually to see errors python scripts/review_dashboard.py :: Check browser developer console (F12) for JavaScript errors :: Clear browser cache: Ctrl+Shift+Delete

5 Windows Service Fails to Start (Error 1053)

Symptom: Service shows error 1053 (timeout)

:: Run manually to see the actual error python scripts/review_dashboard.py :: Check for port conflicts netstat -ano | findstr :8787 :: Reinstall the service python installer/service_wrapper.py remove python installer/service_wrapper.py install sc start LedgerLinkAI

6 Setup Wizard Will Not Open

Symptom: http://127.0.0.1:8790/ does not load

:: Check if the wizard process is running netstat -ano | findstr :8790 :: Start manually python scripts/setup_wizard.py

7 Cloudflare Tunnel Not Connecting

:: Check service status sc query cloudflared :: Restart the service sc stop cloudflared sc start cloudflared :: Check logs type %USERPROFILE%\.cloudflared\cloudflared.log

8 License Key Activation Fails

Cause: Signing secret mismatch or expired key

  • Verify the .env file contains LEDGERLINK_SIGNING_SECRET
  • Check the key has not expired (expiry date encoded in the key)
  • Contact the administrator to regenerate the key if needed

9 Email Test Fails

See Module 13 for SMTP troubleshooting. Common causes: wrong App Password, wrong port, firewall blocking 587.

10 WhatsApp/Telegram Webhook Not Receiving

Verify: Cloudflare tunnel is running, OpenClaw webhook URL ends in /ingest/openclaw (not /whatsapp/webhook), and the LedgerLink service is running. Check messaging_log table for unknown_sender entries.

11 Database Locked Error

Symptom: "database is locked" error

:: Stop all processes accessing the database sc stop LedgerLinkAI :: Wait 5 seconds, then restart sc start LedgerLinkAI

12 Antivirus Quarantines Python Files

Add exclusions for: C:\LedgerLink\, C:\Program Files\Python3*\, and the %APPDATA%\Python\ folder.

13 Mac Gatekeeper Blocks the Installer

System Preferences → Security & Privacy → General → click "Allow Anyway" for the blocked script.

14 Low Disk Space Warning

:: Check disk space wmic logicaldisk get size,freespace,caption :: Clean old backups (keep last 5) :: Delete files in data\backups\ older than 30 days

15 Memory Error on Startup

Cause: Not enough RAM. Check system memory:

:: Windows systeminfo | findstr Memory :: Close unnecessary applications and restart the service
